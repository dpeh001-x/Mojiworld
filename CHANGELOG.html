Phase 1 Foundation

Save / load — versioned localStorage

  • saveState() / loadState() / clearSave() / hasSave() with explicit field whitelists.
  • SAVE_VERSION = 1 gate — future schema changes either migrate or fall back cleanly.
  • Autosaves every 15 s of wall time, plus on class-select, level-up, boss-kill, respawn, tab-hide, and beforeunload.
  • Continue-from-save always lands in town with clean transient state — never loads into hp ≤ 0 or a mid-boss killbox.
  • Phase-2 placeholder fields (achievements, dailyState, prestige, comboRecord, muted) pre-added to the schema so follow-up features don't need a version bump.

Web Audio — synthesized, no assets

  • 4 SFX via AudioContext oscillator envelopes:
    • Hit — 240 Hz square blip
    • Crit — 520 / 780 Hz two-tone
    • Level up — C-E-G triangle arpeggio
    • Death — 180 Hz sawtooth slide
  • Initialized on first user gesture (class-select click, keydown, or pointerdown) to satisfy Chrome/Safari autoplay policy.
  • M toggles mute. Mute state persists in the save.

Phase 2 Retention hooks

🏆Achievements + codex (press N)

  • 12 milestones tracked in game.achievements, persisted via the Phase 1 schema.
  • Modal shows unlocked + locked rows, live kill counter, and best-ever combo (game.comboRecord).
  • Hooks fire from killMonster, the level-up loop, bumpCombo, and attemptEnhance.

🌅Daily login + rotating challenge

  • Deterministic UTC-day rotation — Math.floor(Date.now() / 86400000) % 4. Four challenges: 50 kills, reach level 15, enhance once, 30-hit combo.
  • Login bonus: +200 × min(7, streak) Lumens.
  • HUD banner (top-right) shows the current challenge, progress bar, and streak.
  • Rolls on new-game class-select and on every load.

⟳Respec (button inside the attributes modal, U)

  • Cost formula: 2000 × max(1, level − 5) Lumens.
  • Refunds all allocated AP to skillPoints and resets milestonesUnlocked.
  • Live cost + "you have X" preview rendered every frame while the modal is open.

✨Prestige / Ascension

  • Auto-offered when the player reaches level 50.
  • Each ascension: +30 % XP, +30 % damage, +1 permanent starting AP — all stackable.
  • Resets level / exp / attributes / job / master / inventory / defeated-boss set.
  • getAtk() applies game.prestige.dmgMult. All XP gains (normal + super-boss) apply game.prestige.xpMult.

⚡Combo milestone buffs (inside bumpCombo)

  • Cross 50 hits → +50 % ATK for 8 s.
  • Cross 100 hits → next 3 crits guaranteed.
  • Cross 150 hits → +100 % XP for 10 s.
  • Timer buffs decay via the existing player.buffs tick loop. The guaranteed-crit counter is tracked on player._guaranteedCrit so it doesn't decay as a timer.

Phase 3 Juice pass

Per-hit impact flash

Every swing now fires an extra screen flash — flash(0.1) on basic hits, flash(0.35) on crits — on top of the existing shake / hit-stop / audio / particles.

Damage-scaled feedback

  • Shake now scales with finalDmg: addShake((isCrit ? 5 : 1.8) + min(6, finalDmg/200)).
  • Damage-number font size scales with damage for both normal and crit, so a 500-damage crit visibly towers over a 40-damage jab.

Drop-rarity theatrics

  • emitRarityBurst(x, y, rarity) fires rarity-keyed bursts on every loot site (normal kill, mini-boss, boss extras, super-boss pile):
    • Rare 14 blue particles + small flash.
    • Epic 24 purple particles + flash(0.18) + addShake(2.5).
    • Legendary 40 gold particles + flash(0.35) + addShake(5) + level-up chime.
  • emitNearMiss(x, y) teases 6 gray sparkles when a drop roll failed by < 4 % margin.

Enemy squash-stretch on hit

  • hitMonster sets m.scaleX / m.scaleY — 1.22 / 0.74 on crit, 1.14 / 0.82 on normal.
  • updateMonsters eases them back to 1.0 at 22 %/frame.
  • The drawMonster call site wraps in ctx.save / scale / restore, anchored at the sprite's feet so the entire monster pops without floating.

Boss phase transitions

  • Aetherion phase 2 + 3 now call addHitStop + audio.play('crit') for cinematic punch.
  • Gravitos (v0.9.x addition) gets the same treatment — phase 2 and phase 3 both carry a hit-stop + audio cue.

Patches Follow-ups

Reset button

Destructive option inside the attributes modal (U), below Respec. Double-confirms, calls clearSave(), and reloads to class-select. Sets game._resetting = true first so the autosave / tab-hide / beforeunload handlers don't rewrite the save during the reload window.

Also surfaced on the HUD as a small red ☠ Reset pill next to the help button (bottom-right), so a friend loading the build can wipe the save without digging into the attributes menu. Reuses the same resetSave() flow. The pill sits at z-index: 60 so it stays reachable even while the class-select modal (.modal-overlay, z-index 50) is open.

Codex keybind fix — C → N

Skills bind to fixed slot letters ['d','s','a','e','w','q','c','x'], so C triggers a skill cast for any class with a c-slot skill. The codex shortcut was firing in parallel with the skill cast, opening the modal mid-combat. Moved codex to N; no collision.

✦Juicy damage numbers

Dopamine-driven pass on enemy damage popups so every hit lands as a tiny reward. All the effects live in drawDamageNumbers() and the main hitMonster push, so secondary pushes (EMPOWERED, ECHO, VOID!, EXEC, etc.) inherit the juice automatically via field back-fill.

  • Pop-in overshoot. easeOutBack scale from ~0.4x to 1x over 8 frames — each number punches onto screen instead of appearing flat.
  • Shrink-and-fade tail. Final ~14 frames shrink while fading so numbers "poof" out cleanly, closing the micro-reward loop.
  • Scatter. Random horizontal vx (with drag) plus a short rotation wobble on spawn so stacked hits don't read as a single column.
  • Tiered colour ramp. Non-crit tiers by damage — <80 white → <200 warm → <500 orange → 500+ hot red. Crits stay gold; 500+ crits shift to amber. Magnitude is readable pre-attentively before the eye parses the digits.
  • Embossed crits. Crits render with a coloured shadowBlur glow and an inner #fff7b0 stroke over the black outline — the "gold foil" look that telegraphs a big hit.
  • Comma formatting. finalDmg.toLocaleString() — 1,234 reads bigger than 1234, and four-figure hits now feel like a milestone.
  • Linger on big hits. Life bumped from 46 → 52 (tier ≥ 2) / 58 (crit) so the player has time to admire the number.

Hotkey 0 — change job / unlock mastery (v0.9.9)

Hotkey 0 now acts as a shortcut to the advancement flow so players don't have to wait for the automatic modal to re-fire.

Master skill bar fix — level-20 X slot

At level 20 every master advancement unlocks an ultimate skill bound to the x slot (Grand Warcry, Apocalypse Swing, Shadow Clones, Meteor Shower, One Shot One Kill, etc.). renderSkillBar was calling skillsForClass(player.cls, player.job) — missing the third master argument — so the filter at skillsForClass silently dropped any skill scoped to a master, leaving the X slot stuck on its locked placeholder. Casting also effectively failed because the player never saw the icon to know it was ready. Same bug fixed in the NPC skill-trainer modal (openSkillsModal).

  • Spawned mobs burst in with a gold/white particle puff (_towerSpawnOne).
  • Wave cap raised to 14 mobs per floor so the streaming model has runway.

v0.25.26 Layered chibi sprite sheets — base body in, equipment slots wired

Reintroduces the layered-sprite-sheet pipeline disabled in v0.25.2, this time as a proper modular system designed for adding equipment incrementally without redrawing the base. The nude chibi base body is on screen now; hair / armor / cape / helmet / shield / weapon / boots slots are wired in but empty — drop a PNG matching the 4×4 pose layout into Sprites/character/<slot>/<id>.png and it appears.

🖼️ The asset

Sprites/character/chibi_nude.png — 1216×2312, 4×4 grid of 304×578 walk-cycle frames. Pre-loaded at boot via _loadCharLayer('base_chibi', ...). Other equipment slots load lazily on first lookup.

🧩 Why layered sprite sheets and not skeletal

The big concern with flat sprite sheets is equipment variety — every armor / helmet / cape would multiply the per-frame asset count. Layered sheets sidestep this: each equipment piece is its own per-pose PNG drawn to match the base's frame layout, transparent everywhere else. Composited at runtime, base + gear reads as one unified character. This is how MapleStory, Stardew Valley, and most chibi-2D MMOs handle the equipment problem — you draw each gear piece once across the same 16 poses, not once per body × armor × helmet × etc combination.

Skeletal animation (Spine / DragonBones) was the alternative — lower per-piece artist cost, but requires rigging the base, paid tooling, and weeks of pipeline work before anything ships. The layered-sheet path lets you ship the base today and add equipment one piece at a time without restructuring.

🛠️ How equipment will work

  1. Artist draws a gear piece across all 16 poses, matching the base sheet's pivot points (transparent where no gear).
  2. Save as Sprites/character/<slot>/<id>.png (slot ∈ hair / armor / cape / helmet / shield / weapon / boots).
  3. Set the relevant equipped item's spriteKey to '<id>'.
  4. Item appears on the character with zero code changes.

📐 Architecture

  • CHARACTER_ASSETS — module-level Map keyed by '<slot>_<id>'. Lazy-load on first access via _loadCharLayer; missing files silently no-op.
  • CHAR_FRAMES_X / CHAR_FRAMES_Y = 4 × 4 — frame grid layout, configurable per future sheet.
  • _currentCharFrame() picks frame index from player state. v0.25.26 implements idle (frame 0) + 4-frame walk cycle. Future iterations extend this with attack / jump / hurt rows.
  • _drawCharFrame(img, sx, sy, frame, facing) blits one frame of one layer with horizontal-flip support. Anchored to the player's feet so the figure stands on the ground regardless of source frame height.
  • drawLayeredCharacter orchestrates the layer stack: base → hair → armor → cape → helmet → shield → weapon → boots. Returns true on successful base draw so drawPlayer skips procedural fallback.

🔄 Fallback path preserved

drawPlayer's existing dispatch is unchanged: it tries drawLayeredCharacter first, falls back to procedural drawCuteCharacter if anything fails. If you delete the base PNG or the network blocks it, the game still renders via the procedural pixel art that's been there since v0.25.2.

⚖️ Known scope trades

  • Procedural face overlay still draws on top. The sprite has no facial details; _drawCuteFaceOverlay brings the character to life. If the eye position is off vs the sprite head, future polish can add face frames as a separate layer.
  • Frame layout assumed 4×4. If your future sheets use a different grid (e.g. 8 columns × 2 rows for a more granular cycle), tweak CHAR_FRAMES_X / Y or move to per-asset metadata.
  • Walk cycle uses row 0 only. Other rows in the current sheet may be different states — adjust _currentCharFrame() when you know the layout.
  • Procedural cape / weapon swing animations skipped when the sprite path is active. They were calibrated for the procedural body; sprite-based gear handles those visuals once the gear PNGs land.

  • Elite + mini-boss auras → off-screen canvas cache. Both aura types used to build a fresh createRadialGradient every frame for every on-screen elite/miniboss. First time any unique size appears the aura renders ONCE into an off-screen canvas; every subsequent frame just blits via drawImage. Pulse animation moves from varying the gradient radius to modulating globalAlpha (+/- 8-10%) — visually near-identical, hugely cheaper. On maps with 3+ elites on screen the framerate win is noticeable.
  • Nameplate measureText cached. The peer render loop called ctx.measureText(name) once per peer per frame — measureText forces font-metrics recalculation. Same bug on the local _drawPlayerNameTag. v0.24.0 Multiplayer — real WebSocket server

First step toward a real MMORPG. A Node.js WebSocket server lives in server/; the client gets a 🌐 Multi button, presence tick, ghost-peer rendering, emote relay, and in-world chat. Everyone on the same room name sees each other run around and can chat. Combat / mobs stay per-client for this milestone — that's authoritative-lite, not full MMO.

server/ — Node.js WebSocket relay

  • Single-process server.js using ws. HTTP wrapper so /health returns version + room/player counts for load balancers.
  • Room-based — any unique room id, default lobby. 16 players per room hard cap, 60 s idle kick, 40 ms per-player state-tick floor, 200-char chat cap, control chars stripped.
  • Protocol: hello / state / chat / map / emote (client → server); welcome / joined / left / state / chat / emote / error (server → client). All JSON. Full table in server/README.md.
  • No database, no auth, no persistence — rooms die when the last player leaves.
  • Deployment: Dockerfile included, Fly.io / Railway / VPS instructions in server/README.md. Fly's free tier covers a single small instance.

Client — net module

  • 🌐 Multi button top-right of the HUD opens the multiplayer modal. Fields: server URL, your name, room id. All three persist in localStorage.
  • Presence tick — _mpTick() runs after updatePlayer and sends a state frame every 70 ms (~14 Hz): x/y/vx/vy, facing, map, hp/mp, level, cls/job/master, anim.
  • Peer rendering — _mpDrawPeers() draws each peer in the current map as a translucent violet silhouette with name + level + HP bar above their head. Chat + emote bubbles float above them.
  • Emote relay — pressing 4-8 emotes to everyone in the room.
  • Chat — press Enter while connected to open a bottom-left input. Enter sends, Esc cancels. Messages show as speech bubbles above the sender for 4 s and in a rolling log of the last 30 lines.

What it IS

  • Presence + chat across the room — friends running around in the same map.
  • Per-map scoping — peers on different maps are hidden.
  • Full stat broadcast — HP, MP, level, class, advancement all mirror to peers.
  • Free to run locally or on any tiny VPS / Fly.io / Railway instance.

What it is NOT (yet)

  • Not authoritative combat. Every player instances their own mobs.
  • No shared loot, no trade, no party XP, no PvP.
  • No accounts / persistence. Reconnect = new id.
  • No anti-cheat — server trusts client state. Fine for co-op, not competitive. Server-side combat is the next phase.

v0.18.0 Class Identity — four distinct playstyles

Every class now plays meaningfully differently after job advancement (Lv 10+). Existing skills stay the same — a new mechanical layer wraps around them so how you cast starts to matter as much as what you cast.

⚔ Warrior — Charge & Release

  • Hold a skill key to charge. A ring fills around the warrior over ~0.75 s. Release fires the skill with up to +100% damage.
  • At 100% charge: guaranteed crit + hit-stop + screen shake + orange particle burst. Toast reads "⚔ EMPOWERED".
  • Quick-tap (<120 ms) casts at minimum charge — normal damage.
  • Charge cancels if you get stunned or die. Auto-cast polling is suppressed while a key is held, so you don't accidentally spam the skill.
  • Applies to every warrior skill except basic attack — X S C D F V G all support charging.

🗡 Rogue — Momentum

  • Moving builds a gauge (0-100). Running full-tilt fills it in ~1 s. Standing still drains it faster than it builds.
  • Skills spend most of the gauge (−60) for up to +100% damage. A full-gauge cast is a massive burst.
  • Visible as a thin purple bar above the rogue that brightens to pink near 100%.
  • Rewards hit-and-run / corridor-running play. Standing still to channel skills gives 0 bonus.
  • Quick-dash, rush, and dodge roll all count as movement for gauge purposes.

🔮 Mage — Chain Cast

  • Cast two DIFFERENT skills within 1 s → the second one deals 1.5× damage. A floating "⌬ CHAIN" indicator appears above the mage after the first cast so you know the window is live.
  • Every crit shaves 150 ms off every other skill's cooldown. Mage perks already give 30% CDR; this stacks on top — critting into a rotation feels like the cooldowns evaporate.
  • Gated to once per frame so a single AOE tick can't reset everything.
  • Rewards spell-rotation play instead of spamming one nuke.

🏹 Archer — Rooted Fire

  • Casting a skill freezes the archer in place for ~0.4 s (forces vx = 0, gravity still applies). In exchange you get +25% damage.
  • Standing still builds Steady Aim — stacks climb +0.25/frame up to +40% damage. A "◎ AIM +N%" label appears above the archer.
  • Moving horizontally at any speed breaks Steady instantly.
  • Skill damage formula while rooted: base × (1.25 + steady/100). Max boost: +65%.
  • Rewards planting your feet and actually aiming — suits the class fantasy of a sniper who can't move-and-shoot.

How it works under the hood

  • New module CLASS IDENTITY near castSkill. Each class has a _tickClassIdentity pass per frame and a _computeClassSkillMul hook called before every skill effect runs.
  • Central damage multiplier on player._classSkillMul with a 2-second expiry window, so delayed projectiles / tethered volleys from the same cast still pick up the bonus.
  • Applied in hitMonster for non-melee / non-thorns / non-aegis hits.
  • All four states (_warCharge, _momentum, _steady, _rootedTimer, _mageLastCastT) are cleared on death, respawn, loadMap, and loadState. No leaking between runs.

  • Pre-push auto-rebase (.claude/hooks/mobile-pre-push.py). When Claude is about to run git push on mobile-ui-pass, the hook fetches origin/main and rebases first. Blocks the push with a clear conflict list if the rebase collides; exits clean (hook skipped) if the working tree is dirty or the branch is already up-to-date.
  • Zone guard (.claude/hooks/mobile-zone-check.py). After every Edit / Write / MultiEdit on maple_game.html, the hook computes the changed hunks' line ranges and flags any that fall outside the mobile "safe zones" (7–10, 33, 65–322, 472–593, 1135–1155, 1464–1500, 3317–3744). Warning only — never blocks. Safe-zone list documented in CLAUDE.md.

Both hooks no-op on branches other than mobile-ui-pass, so they're safe to keep registered in .claude/settings.json.

How to try it

git fetch origin main
git switch main
open maple_game.html

Or drop the file into any static server / preview. No install step.

Verification checklist

  1. Save. Pick a class, reach level 2, kill a mob, refresh. Expect Welcome back! toast + same level / mesos / map (you'll land in town).
  2. Audio. Hit a mob → tick. Land a crit → bright two-tone. Level up → 3-note rise. M toggles mute.
  3. Daily. Banner top-right shows progress. Complete the challenge → reward toast.
  4. Codex. Press N — 12 achievements, live kill count, best combo. Kill the first mob → First Blood unlocks.
  5. Respec + Reset. Open attributes (U). Respec refunds AP + wipes milestones. Reset (red button) wipes the whole save.
  6. Prestige. Dev console (hold 1+2+3) → set level 50 → kill a mob → ascend prompt. After: L1 with permanent ×1.3 XP / ×1.3 DMG.
  7. Combos. Farm a weak mob past 50 / 100 / 150 hits — three escalating buffs.
  8. Juice. A 500-damage crit should be visibly huge; rare drops glint, legendary drops explode. Aetherion and Gravitos phase shifts hit-stop the frame.
