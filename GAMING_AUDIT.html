2. Context

Three Explore-agents swept the v0.10.2–v0.11.0 codebase in parallel and produced convergent findings. The game now has solid micro-reward fundamentals — tight hit→combo→level feedback, juicy damage numbers, save persistence, achievements, daily challenges, prestige, respec — all of which shipped in an earlier phase and are now baseline.

What remains is the identity + re-engagement layer. Specifically:

  • Dead zone L20→L50. Auto-popup advancement modals guide the first 20 levels perfectly. After L20 the player has no surfaced next goal and must open modals to find one.
  • Silent rewards and idle dead-air. Potion drops, MP refunds, cooldown refunds, chest spawns all happen with zero audio/visual signal. Standing still for >30 s provides no re-engagement cue and eventually results in death without warning.
  • Zero identity surface. player.look holds hair / skin / eye-colour fields but only gender is wired to UI. No character name. No titles. Class / job / master exist as titles but aren't cosmetic.
  • Zero cross-session re-engagement. Tab title never updates. No "daily ready" indicator. Streak break is silent.
  • Weak completionist hooks. 12 achievements exist but no monster bestiary, no item ledger, no local leaderboards.
  • Fixed difficulty ceiling. No NG+, no infinite tower, no hard-mode. Prestige is the only post-L50 loop and is reachable only after 100+ hours.

Intended outcome: push first-week retention from ~40 % attrition back to ~15 % by surfacing long-tail progression goals, adding identity anchors, and filling the silent-reward gaps.

3. Scope

In: changes to maple_game.html, small HTML/CSS additions to HUD and modals, fields added to the existing localStorage save schema (behind a SAVE_VERSION gate with migrate()), audio synthesised via the existing audio module, and a changelog entry per ship.

Out: backend / server / database, multiplayer or real leaderboards, paid push-notification services, new sprite or audio assets, rewriting existing systems.

11. Verification plan

Each ship = one atomic commit = one CHANGELOG.html entry + one "verification:" block in the commit body.

  1. Syntax gate. Extract the <script> from maple_game.html into a temp file, node --check before committing.
  2. Save compat. Load a save from before the change; confirm no throw, no data loss, no unexpected state reset.
  3. Live-run smoke. Open main in a fresh tab and exercise the changed path end-to-end (potion pickup toast, tab title update, bestiary silhouette → icon, etc.).
  4. Regression spot-check. Land 10 hits on a slime + 1 crit on Gelmonarch. Expect damage numbers, combo, audio, no console errors.
  5. Cross-browser. Open the raw.githack preview URL in Firefox (or mobile) once per phase.

End-to-end check per phase:

  • After Phase 1: the first 10 minutes feel noticeably "louder"; HUD shows a next goal; tab title hints at return.
  • After Phase 2: bestiary + name + titles make the first hour feel personal; streak grace cuts daily-retention bleed.
  • After Phase 3: Shardfall Tower + phase bosses give the L50+ player something to do for weeks.

12. Psychology-lever glossary

Short definitions with canonical examples — so anyone reading this cold understands the terminology.

1. Variable-ratio reinforcement
Rewards delivered on an unpredictable schedule. The slot-machine effect. Example: a 12 % chance of a rare drop per kill keeps players engaged far longer than a guaranteed drop every 10 kills, even at identical expected value.
2. Loss aversion
Losses weigh roughly twice as much as equivalent gains. Example: a visible "streak: 6 days" that a player can break hurts more than the streak bonus helps — so warnings + grace periods retain more than raw rewards.
3. Completion bias
Unfinished collections create a pull to close the gap. Example: a bestiary showing 23/24 silhouettes drives exploration toward that one unseen mob, even if the mob's reward isn't special.
4. Social proof
Evidence that others are invested makes investment feel safer. Example: "your best: 187 combo · world best today: 230" encourages the player to push further — even a local-only best ("yesterday you: 187") works.
5. FOMO (Fear of missing out)
Time-limited content creates urgency. Example: a daily challenge that resets at midnight, or a 30 min "lucky hour" with 2× loot, pulls players back before the window closes.
6. Endowment effect
Things a player personalised or named feel more valuable. Example: a character called "Kael" with custom hair is much harder to delete than an unnamed default — even if mechanically identical.
7. Progress overview
A unified, always-visible answer to "what's next?". Example: a HUD widget showing "Next: Centurion (87/100 combo)" collapses uncertainty and keeps the player pointed at a goal.
8. Near-miss reinforcement
Almost-winning is neurochemically close to winning. Example: XP bar glowing in the last 10 %; "so close" teaser at 90 % of an achievement. Engages dopamine even on non-completion.
9. Re-engagement
Cues that pull the player back into the game from outside it. Example: browser tab-title updating to "✦ Daily ready — LevelX" while the tab is backgrounded in another window.
10. Mastery curve
Challenge escalates in lock-step with player skill, so the game is never trivial and never impossible. Example: infinite tower modes, NG+, or per-boss phase patterns that demand new tactics.
11. Flow state
A tight action-feedback loop the player gets lost in. Example: hit → combo-tick → damage number → crit → hit-stop — all inside 120 ms. Any pause (silent idle, dead-air after a kill) breaks flow.
12. Identity / self-expression
Players stay longer when the game lets them say something about themselves. Example: unlockable titles ("The Ascendant"), cosmetic items, a chosen name and look. Earned identity > granted identity.
