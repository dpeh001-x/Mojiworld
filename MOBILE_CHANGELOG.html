HUD overlap fix bug

Desktop HUD corners no longer collide with mobile thumb zones

The desktop HUD buttons (#version-tag, #minimap, #taxi-btn, #help-btn, #reset-front-btn) are absolute-positioned to the corners of .game-wrapper. On mobile those same corners are where the on-screen touch controls live — so the two layers were blocking each other's taps and stacking visually.

  • Hidden on mobile: #minimap (220×74 — too big to fit), #taxi-btn (redundant; the in-game menu button exposes taxi).
  • Moved to a tight top-LEFT utility column: #version-tag, #help-btn, #reset-front-btn — all out of the thumb reach zones.
  • Relocated: #mobile-ctrl-toggle nudged from top-right (stacking with potions) to just below the potion row so it stops overlapping the ♥ / ◆ buttons.
  • All rules scoped to @media (pointer: coarse), (max-width: 900px), inside the existing Mobile Controls (v5) CSS block — so desktop is untouched.

Verified via the mobile-zone guard hook: every change lands in line range 65–322 (the MOBILE CONTROLS (v5) safe zone) so there is no conflict risk with the collaborator's gameplay work on main.
