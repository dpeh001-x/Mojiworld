Executive summary one-pager

LevelX on mobile is mechanically solid but feedback-starved. Combat, controls, scaling, and modal tappability all work after recent fixes. What's missing is the thin layer of visible progress and reward that keeps a phone player engaged between kills. Three independent Explore agents — one per theme (rewards / progression / retention) — identified the same root cause from different angles.

  • Rewards are silent on mobile. XP gain, skill-point earns, boon pickups, combo milestones fire without toast or HUD animation. Particles spawn at monster death coordinates and often land behind the d-pad / face-button clusters.
  • Progression is modal-gated. Job / master / skill-tree / milestone / bestiary / achievement progress all require opening a menu. On mobile every modal is a 2-tap cost — and none of these systems surface a "next milestone" on the HUD.
  • Retention hooks don't exist. No tab-title updates, no localStorage return-hints, no PWA manifest, no daily countdown timer. Once the tab closes, the player has zero pull back.

Estimated impact of shipping the Phase-1 bundle: ~10–15% reduction in first-hour bounce (per Agent C). None of the Phase-1 items require gameplay-code changes — every one lives inside a mobile safe zone.

Cross-confirmed gaps 8 items · all 3 agents

Gaps flagged independently by ≥2 agents. Every claim anchored to maple_game.html line numbers.

G1 Level-up fanfare loud, skill-point gain silent

maple_game.html:10706–10744line 10724 — player.skillPoints += 1

gainExp() spawns 30 particles + DOM .level-up element + audio.play('levelup'), but no toast for the +1 SP that just became spendable. On mobile, menu access is a 2-tap cost — the player won't make it without a prompt.

G2 XP bar fills silently between kills

killMonster:10550–10636

No dedicated feedback — only the HUD bar increments. On a 320px iPhone viewport the bar is ~200px wide; incremental gains are sub-pixel for several kills.

G3 Particles and damage numbers spawn under mobile buttons

particle emit: m.x + m.w/2dmg: player.x + 14, player.y - 14

D-pad zone: canvas X 0–156, Y 404–560. Face buttons: X 776–960, Y 376–560. Combat commonly happens in the bottom quadrants; feedback vanishes behind translucent button overlays.

G4 Every progression system is modal-gated on mobile

JOB_ADVANCE_LEVEL:2150MASTER_ADVANCE_LEVEL:2151#daily-banner:1577 (display:none default)

Job advancement countdown (invisible pre-Lv 25), master/prestige race at 50, skill-tree AP cue, milestone unlocks on stat thresholds, bestiary cosmetic-unlock counters, achievement progress — none surface a "next up" on the HUD. Even the daily banner defaults to hidden.

G5 No re-engagement layer exists

static <title> line 5no manifest.jsonno navigator.serviceWorker

document.title never updates. No PWA, no service worker, no Notification permission flow. Rotate-nag dismissal isn't localStorage-persisted (line 3767–3769) — resets every refresh. Result: a closed tab is a lost player.

G6 Death overlay is passive and unresponsive

#death-overlay:1030–1066

pointer-events: none, no respawn button, 1.6s fade. On mobile this feels like input frozen — breaks flow on every death.

G7 Silent pickups

spawnPowerupOrb:5152enhance fail toast:11661

Powerup orbs play no sound and fire no toast on collection. Enhancement failure fires a plain toast with no rarity color — visually identical to any common notification.

G8 Daily reset countdown invisible

dailyIndex():4595

UTC midnight boundary. Nothing on mobile HUD tells the player when it flips. Combined with G5: a player with a 3-day streak has zero mobile signal of "come back in 4h to preserve it."

Phase 1 — Quick wins < 1h each · 10 items

Each ships as an atomic commit. All land inside mobile safe zones — see Critical files below.

P1.1 Tab-title badge on level-up S 🟢 high

Re-engagement +3Completion +1JS safe zone 3612–3797

On each gainExp level-up, push document.title = '⭐ Lv ' + player.level + ' — LevelX';. On visibilitychange hidden → swap to attention-pulse form; on focus → revert.

Why: the browser tab is the #1 retention surface for web games. Zero cost, no new save fields.

P1.2 Skill-point toast on level-up S 🟢 high hard-rule flag

Near-miss +2Completion +2

Clean fix — 2 lines at maple_game.html:~10751 after audio.play('levelup'): showToast('+1 Skill Point — open Skills (I)', 'rare'). That's gameplay code.

Safe-zone alternative: in mobile JS safe zone, poll player.skillPoints every ~500ms and fire toast on delta. Slightly indirect but safe-zone-local. User decision needed before ship.

P1.3 Rotate-nag dismissal persists across reloads S 🟢 high

Quit-point removalJS 3767–3769

At lines 3767–3769, replace single-dataset write with dataset + localStorage.setItem('_mobile_nagDismissed', '1'). On nag-evaluation path, read localStorage first and early-return.

Why: currently re-nags every page refresh — annoying for repeat players.

P1.4 XP-bar near-level pulse S 🟢 high

Near-miss +3CSS 65–334JS 3612–3797

When player.exp / player.expToNext > 0.9, set body.xp-near. CSS media-gated rule applies 0.8s pulse-glow animation on the XP fill element.

Why: the last 10% of any bar is the Casino rule — visible "almost there" drives one-more-kill.

P1.5 Daily-banner visible + countdown S 🟢 high

FOMO +3Loss +2#daily-banner:1577

CSS override on mobile: @media (pointer: coarse) { #daily-banner { display: block !important; } }. JS tick (once/minute) appends … resets in 5h 32m using Date.now() vs next UTC midnight.

Why: the banner already has a progress bar and streak counter. Mobile just hides it.

P1.6 Route damage numbers out of bottom-right S 🟡 medium hard-rule flag

Flow +2

On mobile, clamp damage-number spawn Y to ≤ 340 when the player is in the lower-right quadrant. Cleanest fix is at the combat-code emit site (gameplay). Safe-zone alt: mobile JS MutationObserver on the damage-number container that repositions nodes after append. User decision needed.

P1.7 Haptic vibrate on legendary drops S 🟡 medium

Variable-ratio +2JS 3612–3797

MutationObserver on toast DOM nodes with .rarity-legendary / .epic → fire navigator.vibrate([60, 40, 60]). Zero gameplay touch.

Why: haptic is a mobile-only feedback channel the game doesn't use yet.

P1.8 Streak-at-risk toast on load S 🟢 high

Loss +3Re-engagement +2

On page-load, if player.dailyState.streak > 1 AND Date.now() + 2*3600*1000 > nextUTCMidnight, show toast ⚠ 2h left to keep your N-day streak and pulse the daily banner 3s.

Why: turns passive streaks into loss-aversion cues (~2× the pull of gain).

P1.9 Next-advancement chip in HUD S 🟢 high

Progress overview +3DOM 1476–1512

Small badge in mobile HUD. Content: Lv < 25 → → Job in N; 25–49 → → Master in N; ≥ 50 → → Ascend in N or hide. One-line JS tick to refresh on level-up.

Why: removes the 25-level flat zones between advancement tiers.

P1.10 Death-overlay tap-to-respawn S 🟢 high hard-rule flag

Flow +2#death-overlay 1030–1066

Mobile-only override: add pointer-events:auto on an inner element, attach click that speeds up respawn. Depends on whether respawnAtTown exposes a safe accelerator. If not, needs a gameplay-code helper. User decision needed — want me to read the respawn code first?

Phase 2 — Structural additions 2–4h each · 6 items

P2.1 Unified "Next Goal" mobile HUD widget M 🟢 high #1 ROI

Progress +4Completion +2Near-miss +2Loss +1

A single 28–40px pill in the top-right stack that cycles (~5s) through the player's nearest milestone: next job/master level, next achievement (by lowest remaining %), next cosmetic unlock, nearest daily quest goal, nearest skill-tree unlock. Each entry shows → Target (X of Y).

Data sources (all existing — no save migration): player.level vs JOB_ADVANCE_LEVEL / MASTER_ADVANCE_LEVEL; game.achievements vs ACHIEVEMENTS; game.bestiary vs COSMETICS (lines 11843–11862); player.dailyState. Pure read layer.

Why: Agent B's biggest gap. Pulls the 4 weakest levers simultaneously. Highest ROI single item in the audit.

P2.2 Achievement / bestiary inline progress M 🟢 high

Completion +3Near-miss +2trackPickup:11867

After monster death, peek ahead in achievement + bestiary tables to see if this kill contributed to a goal within 5 of completion. Subtle toast: 🦂 9/10 — one more for Ember Cape!

Implementation: periodic delta-check on a mobile JS tick (polling once/sec). All reads, no writes.

P2.3 First-run mobile coach marks M 🟢 high

Onboarding conversion

After the class-select modal closes for the first time, overlay 3 pointer arrows (one per mobile button cluster): ← move, attack, menu ↑. Each auto-dismisses on first relevant button press or 5s timeout. Never shows again (localStorage flag).

Why: Agent C estimates ~10% of new mobile players bounce before learning controls.

P2.4 Streak-at-risk notification + countdown M 🟡 medium

Loss +3FOMO +3Re-engagement +3

Extends P1.5 + P1.8. Adds: if browser Notification permission is granted, fire a push ~1h before UTC midnight when streak > 2. Escalates passive banner → pulsing banner → notification → full-screen overlay on next load.

Note: Notification.requestPermission() gated behind an explicit opt-in button in the daily banner.

P2.5 Haptic + audio on silent pickups M 🟡 medium

Variable-ratio +2Flow +1

MutationObserver on the boon inventory array; when a new entry appears, fire navigator.vibrate(40) + audio.play('pickup') (reuse existing sound). Catches gap G7 (silent pickups).

P2.6 PWA manifest + install prompt M 🟢 high

Re-engagement +4

Add manifest.webmanifest (new file at repo root) + <link rel="manifest"> in <head>. Minimal service worker that caches the single HTML; enables home-screen install.

iOS: no auto-prompt but share-sheet → "Add to Home Screen" produces a standalone PWA. Android Chrome: auto-prompts.

Phase 3 — Big bets 6h+ each · 3 items

P3.1 Mobile "Companion Panel" L 🟢 high

Progress +4Endowment +2Identity +2Completion +3

Slide-up panel (accessed via a new ☰ button in the mobile HUD) that consolidates all progression on one screen: character name + look, level + XP, skill points waiting, next advancement ETA, 3 nearest achievements, 3 nearest cosmetic unlocks, daily quest + streak + countdown, recent PBs (combo, tower, super-boss clear time), cosmetics owned.

Why: collapses 5 different menu modals into a single thumb-friendly view. Desktop keeps menu keys.

P3.2 Rotating visibility-change tab title L 🟡 medium

Re-engagement +3Social proof +1

Rotating tab-title system that periodically swaps to the most urgent pending action: ⭐ Lv12 → ✨ +1 SP ready → 🔥 3-day streak → ⚔ Boss ready → cycle. Uses document.visibilityState === 'hidden' to switch into "attention mode" with animated ellipsis.

Why: most mobile players have LevelX in a background tab. The single re-engagement surface that works on iOS Safari without Notification permission.

P3.3 Mobile share card on new PB L 🟡 medium

Social proof +3Identity +2

On boss kill or new PB, capture a 1600×900 canvas snapshot via canvas.toBlob, overlay hero name + stats + kill time, offer the native share sheet via navigator.share({ files: [...] }). Desktop fallback: download.

Why: free social proof — every shared card is organic acquisition.
